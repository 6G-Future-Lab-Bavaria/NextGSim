Dataflow

class sixgedgesim.dataflow.DataFlow(name, activation_distribution=None)

A DataFlow algorithm controls how the message generation of the sensor modules is associated in the nodes of the topology.This assignment is based on a generation controller to each message. And a generation control is assigned to a node or to several in the topology both during the initiation and / or during the execution of the simulation.

Args:

name (str): associated name

activation_distribution (): a distribution function to activate the run function in execution time

add_data_sink(data_sink_description)

Adds a data sink to the data flow that is described in the ‘sink_description’ as a dictionary.

Args:

data_sink_description(Dictionary) : description of the sink module ,

(e.g {“model”: “actuator”, “number”: 1, “module”: app.get_sink_services(), “user”: user_id} )

add_data_source(data_source_description)

Adds a data source to the DataFlow that is described in the ‘source_description’ as a dictionary.

Args:

data_source_description(Dictionary) : description of the source module ,

(e.g {“model”: “sensor_1”, “number”: 1, “message”: app.get_message(“Sensor_1_Data_ACQ_APP1”), “distribution”: d_distribution, “user”: user_id} )
