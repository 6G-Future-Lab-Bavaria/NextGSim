Core

class sixgedgesim.core.Sim(topology, logger=None, results_path=None)

This class contains the discrete-event simulation environment that is responsible for orchestration of the simulation.

Args:

topology (Topology) : Topology of the network that is being simulated. results_path (string) : Path of the folder where results will be saved. logger (Logger) : Logger settings for the simulation log.

Sim.__report_backhaul_status()

Reports information about the messages in backhaul such as : process id of the process they belong, ID of the user they serve, ID of the entity that the message is being or going to be processed, completion percentage of the processing, latency experienced by the app in the backhaul, and a boolean value indicating if the message is waiting to be scheduled by a base station or not ( if this value is None, the message is not meant to be sent by a base station ).

It can report them as a log, or if enabled, it can output them as a .csv file to be used by the RAN simulator.

e.g :

Location : 3 - base_station

[‘Latency’, None, 12.800000004062326, 12.800000008062284]

[‘Waiting to be Scheduled’, False, True, True]

Sim.__perform_computations()

This function executes the processing happening in the simulation environment. COMPUTING_PERIOD parameter of Sim object is used to determine how often computation progress is updated.

Decreasing the computing period increases the granularity of the simulation while increasing the time of simulation.
